# Gemini batch question assistant

🚀 这是一个专门为 Google Gemini 用户设计的自动化工具，旨在解决网页端无法原生批量提交问题的不便。通过在页面注入直观的控制面板，你可以轻松管理提问队列，实现无人值守的连续作业。

### Core features

🌟 脚本内置了智能状态检测机制，能够实时监控 Gemini 的生成状态，确保在前一个任务彻底完成且系统处于空闲时才发起下一次输入。它支持自定义的分隔符，默认使用三个连续的逗号作为分割标记，有效避免了正文中普通标点导致的误判。控制面板提供任务追加、中途插入、删除以及清空等完整的队列管理功能，同时具备物理冷却等待时间以模拟真实人工输入频率。

它的使用场景是，当你要翻译一篇论文，如果一整篇上去，因为上下文的限制，回答的质量会很差。往往需要问完一段内容之后，再给出一段内容，来保证提问的质量。有了这个插件，你就可以一次性把你所有问题都准备好，开始执行后它会以串行的方式去执行你的多个问题。喝杯咖啡站起来活动活动出去吃个饭，你就可以看到整篇论文的翻译结果。其实更理想的方法是通过API来实现并行，但是这个可能API成本更高。

### Installation guide

🔧 你需要先在浏览器安装 Tampermonkey 扩展插件。点击 GitHub 仓库中的 `script.js` 文件并选择 Raw 模式，或者直接复制源码新建脚本。确保你的浏览器访问地址匹配 `https://gemini.google.com/*` 即可自动激活工具栏。

### Usage instructions

📖 点击输入框左侧的列表图标即可唤起控制面板。在文本框输入多个问题并使用 `, , ,` 或者 `，，，` 进行分割，点击回车键或者尾部追加按钮将任务送入队列。点击开始执行后脚本会自动处理后续流程，你可以在状态栏实时查看当前的进度，例如输入中、冷却等待或者已完成等状态。

### Technical details

🛠️ 脚本采用纯原生 JavaScript 开发，利用 MutationObserver 动态监听 DOM 变化以保证在页面路由跳转后依然能准确挂载。输入模拟层通过构建虚拟 Paragraph 元素并触发 Input 事件来绕过编辑器的底层限制，从而实现与官方 UI 的无缝兼容。
